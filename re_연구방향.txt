    - 문제를 해결하거나 해를 찾는 과정에서 어떻게 안정적이고 정확한 계산을 할 수 있느냐가 관건.

>>어떤 연구를 참고할까?
    1. LinCode – 선형 부호의 컴퓨터 분류 알고리즘 (Sascha Kurz, 2019)
        격자점 열거 기반의 선형 부호 분류 소프트웨어인 LinCode의 알고리즘 원리를 설명하고, 
        대규모 부호 집합에서 동형 부호(Isomorphic copies)를 효율적으로 제거하는 기법을 제안
        수억 개 이상의 부호를 분류하기 위해 부호의 가중치 분포에 따라 작업을 분할하는 병렬 처리 구조를 설계
        이 연구는 기존의 'Q-Extension' 등의 도구와 결과를 검증하고, 10^9 단위의 방대한 부호 집합도 컴퓨터로 분류 가능함을 입증

    2. Parameterized Algorithms on Integer Sets with Small Doubling (작은 배가 상수를 가진 정수 집합의 매개변수화 알고리즘)
        기존에 존재론적이었던 프라이만 정리를 시간 내에 작동하는 효율적 구성 알고리즘으로 탈바꿈시킴.
        Ruzsa의 모델링 보조정리, Bogolyubov 보조정리, 격자 기반의 GAP 생성 기법 등을 알고리즘적으로 구현
        배가 상수가 고정된 경우 Subset Sum 문제를 ~~시간에 해결할 수 있음을 보였으며, 이는 기존의 NP-난해성을 극복하는 매개변수적 접근

    3. Projective divisible binary codes(사영 가분 이진 부호 연구)
        모든 부호어의 무게가 로 나누어지는 가분 부호 중에서 열들이 사영적으로 독립된 '사영 부호'의 존재 여부를 조사하고, 
        이를 통해 부분 스프레드의 상한선을 도출
        아몰라 이건 하지마

    4. The Complexity Landscape of Decompositional Parameters for ILP (ILP를 위한 분해 매개변수의 복잡도 지형)
        정수계획법(ILP)의 제약 행렬이 가진 그래프 구조적 특성(Primal graph)을 활용하여, 
        문제의 변수 도메인 크기에 제한을 두지 않고도 해결 가능한 새로운 고정 매개변수 가해(FPT) 조각을 찾는 것
        제약 행렬의 변수들 사이의 관계를 나타내는 primal graph에서 **트리 깊이(Treedepth, TD)**라는 매개변수에 주목
        트리 깊이와 제약식의 최대 계수 ℓ이 고정될 경우, ILP가 FPT임을 증명
        연구의 핵심 알고리즘적 기법은 생략(Omitting)
        트리 깊이 분해 트리에서 서로 동일한 제약 구조를 가진 하위 트리들을 찾아내어 하나만 남기고 제거함으로써, 전체 변수의 개수를 매개변수에만 의존하는 상수로 축소
        변수 간의 동등성(Equivalence)을 정의하고 이를 사전적으로 검사하는 로직을 설계

    5. Computer Classification of Linear Codes Based on Lattice Point Enumeration and Integer Linear Programming(격자점 열거와 ILP 기반의 선형 부호 컴퓨터 분류)
        격자점을 모두 열거하기 전에 ILP 솔버를 호출하여 해당 매개변수의 실행 가능성(Feasibility)을 먼저 검사함으로써, 불필요한 탐색을 사전에 차단 (phase0)
        디오판토스 방정식 시스템을 만족하는 격자점을 열거하여 부호 확장 후보를 생성 (phase1) / 동형성 검사 및 추가 제약 조건을 적용 (phase2)
        '정규 길이 확장(Canonical length extension)' 조건과 '무게 스펙트럼의 갭(Gap)' 조건을 ILP 수식으로 선형화하여 삽입
        이 방법을 통해 100만 개 이상의 후보 부호를 효과적으로 필터링하고, 기존에 알려지지 않았던 사영 가분 부호 및 이중 무게 부호의 비존재성을 증명
